Vendor Query Improvement Project - The project objective was to improve quality of the Vendor Query (VQ) process for main Infosys engagement. At starting point operational team was able to operate at level of 95% VQ closed within 24h - below SLA. Project goal was to improve this metrics to 99% VQ closed within 24h.

It was decided to use Six Sigma (6S) approach to improve process to desired level of Sigma and sustain it. My main responsibilities were:

  • Measure current situation,
  • Conduct workshops and analysis with 6S (Ishikawa Diagram, Pareto Analysis, and FMEA for risk) and data mining tools (statistics overview, visualization and predictive model (machine learning) to categorize queries) to define root cause and to understand why it is not working at desired level,
  • Prepare improvement actions and planning to solve the problem,
  • Manage implementation with many small teams responsible for each actions,
  • Report to main stakeholders â€“ Client Operations Head and Operations Managers,
  • Create status report,

Goal was achieved and process is still on desired level (99% of VQ closed within 24h). Delivered project saving calculated as 70K USD.
Vendor Data Fix - Project was related to cleansing of Vendor Master Data for biggest account in Infosys Poland. Project team analyzed current data (obsolete and duplicated vendors), did the profiling (data quality), and process assessment for future process improvement recommendation.

Problem statement:

  • Inactive vendor information,
  • Duplicated entries,
  • Vendor information that is being inputed into fields are inconsistent across kernels,
  • Lack of ownership and oversight on vendor master data,
  • Complex system - 15 SAP kernels

Within 3 months project team reduced number of vendors by 61% from 608k to 254k, found 35k of potential duplicated vendors and measure quality of data set based on 58 different SAP fields.

Additionally during "Process Fix" phase for which I was responsible, project team identified process and system breakdowns, assessed SAP data model and authorization concept. As results almost 2k users were blocked in SAP for VMD changes. List of systems changes requirement, governance structure and process changes guidelines were delivered and presented.
